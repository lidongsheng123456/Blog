## 背景

毕设项目做完了，但是写论文、截图、画图、录演示视频这些事情非常耗时间。我尝试用AI+自动化工具把这套流程跑通，最终实现了从代码到论文成品的大部分自动化。

项目技术栈：Spring Boot3 + Vue3 + MyBatis-Plus + Sa-Token + Element Plus

## 自动化全流程

### 1. 论文文档自动生成

用`python-docx`库，基于学校的Word模板，自动生成完整的毕业设计成果报告书。

核心思路：
- 先分析参考论文模板的样式（字体、字号、行距、页边距等）
- 用Python脚本`generate_thesis.py`按章节结构自动填充内容
- 图片位置先用占位符文本标记，后续批量替换

```python
# 生成论文的核心流程
# 1. 复制模板文件，保留格式
# 2. 按章节顺序写入内容（摘要→绪论→技术→设计→实现→测试→总结）
# 3. 图表位置插入占位符："[IMAGE_PLACEHOLDER: 图X-Y 描述]"
```

### 2. UI页面截图自动采集（Playwright）

用Playwright MCP自动打开浏览器，登录系统，逐页截图。

```javascript
// 使用Playwright自动截图
// 1. 启动浏览器导航到前端页面
// 2. 自动填写登录表单（验证码需手动）
// 3. 逐个访问管理页面并截图
// 4. 保存到 screenshots/ 目录
```

一共截取了28张UI页面截图，覆盖前台首页、注册、登录、后台所有管理模块。

### 3. 代码截图自动生成

后端Java代码不适合直接贴文本到论文里，所以用HTML渲染+Playwright截图的方式生成代码图片。

```
流程：
1. 创建 code_pages.html，用CSS做语法高亮
2. 每个Controller的核心代码放到独立的div里
3. 启动本地HTTP服务器（python -m http.server 8899）
4. 用Playwright对每个div元素单独截图
5. 保存为 fig4-x_xxx_code.png
```

生成了14张代码截图，包括前后台登录、注册、用户管理、角色管理、权限管理、通知管理等模块的核心代码。

### 4. 图片批量插入论文

用`insert_images.py`脚本，根据`IMAGE_MAP`字典把截图批量替换论文中的占位符。

```python
IMAGE_MAP = {
    '图4-1前台页面效果图': 'fig4-1_front_home.png',
    '图4-2前台页面部分代码图': 'fig4-2_front_code.png',
    # ... 共58个映射
}
```

最终结果：**58张图片全部插入，0个占位符剩余**。

### 5. 演示视频自动录制

用Playwright的`recordVideo`功能，自动录制项目演示视频。

```javascript
// record_demo.mjs 核心逻辑
const context = await browser.newContext({
  viewport: { width: 1280, height: 720 },
  recordVideo: {
    dir: outputDir,
    size: { width: 1280, height: 720 },
  },
});

// 自动访问12个页面：
// 前台首页 → 注册 → 登录 → 后台登录(手动输验证码) 
// → 数据中心 → 用户管理 → 角色管理 → 权限管理 
// → 通知管理 → 字典管理 → 操作日志 → 通用查询
```

登录验证码环节用半自动方式：脚本自动填用户名密码，暂停等手动输入验证码，登录成功后自动继续录制剩余页面。

输出格式为WebM，可用ffmpeg转MP4：
```bash
ffmpeg -i demo.webm -c:v libx264 -preset fast demo.mp4
```

## 目前做不到的点（需要进一步开发）

### ER图、实体属性图、系统功能结构图

这些图表每个学校要求的样式完全不一样，目前没办法自动生成为跟学校模板一模一样的效果。

**具体问题：**

1. **系统功能结构图**：有的学校要树形层级结构，有的要模块框图，节点样式（圆角/直角/颜色/连线方式）各不相同
2. **实体属性图（用例图）**：椭圆+Actor的画法、布局方式、连线样式因模板而异
3. **系统E-R图**：有的用传统Chen记号（矩形+椭圆+菱形），有的用Crow's Foot（鸦脚）记号，实体框的填充色、边框、字体都不统一

目前的做法是用HTML/CSS画一个大致的版本插入论文，但跟学校参考论文里的图样式差距明显。

**要真正解决这个问题，需要：**
- 从参考论文中提取图表的视觉风格（填充色、边框色、记号类型、字体大小）
- 针对不同记号类型（Chen/Crow's Foot/UML）开发对应的SVG渲染器
- 支持从SQL建表语句自动提取实体和关系，生成对应风格的ER图
- `thesis-generator`这个skill里有相关的设计思路，但完整实现还需要不少工作量

### 验证码自动识别

登录页面有图片验证码，目前只能手动输入。可以接入OCR或者在测试环境关闭验证码来解决。

|------|-----------|------|
| 论文正文生成 | ✅ 全自动 | python-docx按模板生成 |
| UI截图 | ✅ 大部分自动 | Playwright自动截图，验证码手动 |
| 代码截图 | ✅ 全自动 | HTML渲染+Playwright截图 |

| 降AI检测率 | ⚠️ 有方案未实施 | thesis-generator skill内置6套改写模板 |

整套流程跑下来，从项目代码到论文成品+演示视频大概2-3小时就能搞定，省了非常多时间。
